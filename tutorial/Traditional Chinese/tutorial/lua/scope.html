區域變數
先寫 local 再寫變數名稱然後賦值就是在建立區域變數了
用 lua 內建的 print 函式可以檢查出該區域變數是否被刪除了
被刪除的變數會使得 print 印出 nil


local g = 8   -- 建立區域變數
print(g)      -- 會印出8
g = g+1
print(g)      -- 會印出9
g = nil       -- 刪除此變數
print(g)      -- 會印出 nil









自動刪除區域變數
區域變數不像全域變數需要手動刪除
當時機一到
區域變數會自動刪除自己


區域變數自動刪除的地點有兩個：
	1.遇到關鍵字 end，但不是任何一個 end 都可以
	2.程式碼最後一行的結尾


要說明區域變數遇到哪一個 end 才會自動刪除
而哪一種區域變數要等到最後一行的結尾才會自動刪除
用縮排來解釋是最簡單容易理解的

下面兩個範例只使用 do 跟 end 這兩個關鍵字來造成縮排
do 跟 end 之間的每行程式碼都右移一個 tab 的長度
這就是縮排


local g = 8      -- 下面每一行都讀的到這變數

-- 這裡簡單的用 do end 來造成縮排
do
    print(g)     -- 印出8
    print(x)     -- 印出nil，因為此時 x 不存在
    local x = 4  -- 建立區域變數 x
    print(x)     -- 印出4
end

print(g)   -- 印出8
print(x)   -- 印出ni，因為此時 x 已自動刪除了














只要縮排有整理好
再多層縮排也不會搞混區域變數的刪除時機
下面範例已經清楚的告訴你何時遇到 end 會自動刪除


local x = 1  -- 下面每一行都能讀的到 x 這個值

-- 這裡就只能讀到 x 的值，y z 還沒出現
print(x)     -- 印出 1
print(y)     -- 印出 nil
print(z)     -- 印出 nil

do
    local y = 2
    print(x)     -- 印出 1
    print(y)     -- 印出 2，此時 y 也能讀到值了
    print(z)     -- 印出 nil
    do
        local z = 3
        -- 此時3個變數都是存在的，3個數字都印的出來
        print(x)     -- 印出 1
        print(y)     -- 印出 2
        print(z)     -- 印出 3
    end
    print(x)     -- 印出 1
    print(y)     -- 印出 2
    print(z)     -- 印出 nil，因為已經自動刪除了
end
-- y 也自動刪除了，又只剩下 x 能讀到了
print(x)     -- 印出 1
print(y)     -- 印出 nil
print(z)     -- 印出 nil

























被刪除的變數
那些變數雖然被刪除了
但是它們未必已經把佔用的記憶體歸還給電腦了


Lua 會找個時間歸還記憶體

或者你可以呼叫這個函式來要求 Lua 立刻進行歸還動作
collectgarbage("collect")

有時被刪除的變數積太多會讓歸還行為花很多時間
所以你自己定期去要求 Lua 進行歸還可能會比較好

但是太頻繁的要求進行歸還動作也有可能會浪費更多時間



變數離開了範圍卻還沒被刪除
這在閉包的情況下的確會發生
因為閉包活了下來
被閉包引用的變數因此沒有消失
